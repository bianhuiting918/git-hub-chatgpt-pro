# Project 02: Predicted-TS catalytic-class barrier predictor

Created: 2026-06-28  
Updated: 2026-06-28

## 1. Project goal

This project builds a **predicted-TS student model（预测过渡态 student 模型）** that estimates computed catalytic potential without requiring a true TS as an inference input.

The target setting is:

```text
enzyme sequence/structure + substrate/GS complex + reaction prior
+ TS prediction model output + PLACER screening
→ predict ΔG‡ proxy / ΔΔG‡ proxy / catalytic potential ranking
```

Here:

- GS（ground state，基态）is the bound substrate/reactant complex.
- true TS（真实过渡态）may be available during training through Project 01, but it is **not allowed as a required inference input**.
- predicted TS（预测过渡态）means a TS geometry, TS-like conformer, TS embedding, Δq prior, or reaction-state representation generated by a TS prediction model.
- PLACER is used to generate or screen conformers compatible with the predicted TS prior.
- The first-stage target is computed barrier proxy / catalytic potential ranking, not direct experimental `kcat` or `kcat/KM` prediction.

## 2. Relationship to Project 01

Project 01 is the **true-TS teacher**:

```text
protein + GS + true/refined TS + QM/MM or DFT labels
→ ΔG‡ / ΔΔG‡ / ranking / mechanism components
```

Project 02 is the **predicted-TS student**:

```text
protein + GS + reaction prior + predicted TS embedding/geometry + PLACER screening
→ ΔG‡ proxy / ΔΔG‡ proxy / catalytic potential ranking
```

The intended transfer path is:

```text
Project 01 learns high-confidence true-TS TS-GS scoring
        ↓
Export teacher predictions, true-TS embeddings, barrier labels, and mechanism components
        ↓
Project 02 learns to approximate the true-TS teacher without true TS at inference
```

Project 02 should answer:

> Can a predicted TS prior close the gap between a GS-only model and the true-TS teacher model?

## 3. Scientific hypothesis

A catalytic-class-conditioned model can become more useful if it is not trained as a black-box `embedding → activity` predictor.

Instead, Project 02 should decompose computed catalytic potential into:

```text
1. protein identity / mutation context
2. GS binding and ligand pose
3. predicted TS prior or TS embedding
4. PLACER-screened TS-compatible conformer ensemble
5. catalytic geometry feasibility
6. electrostatic TS-stabilization proxy
7. productive conformer accessibility
8. conformational / ligand strain proxy
```

The main scientific claim is not:

```text
The model directly predicts experimental kcat for all enzymes.
```

The first-stage claim is:

```text
Within a target catalytic type, predicted TS priors and PLACER screening can transfer
true-TS barrier-learning signals from Project 01 to practical inference settings.
```

Experimental kinetics may be used later as optional external validation when assay conditions are comparable.

## 4. Target scope

First-stage generalization should be **within a catalytic type**, not across all enzyme chemistry.

Recommended first catalytic class:

```text
serine hydrolase-like reactions
```

Possible training families:

- PETase / cutinase-like PET hydrolases
- CALB / lipase / esterase benchmark systems
- related serine hydrolases

Later expansions:

- PUase / polyurethane hydrolases
- nylonase and other amide hydrolases
- metal hydrolases
- redox enzymes / laccase
- other catalytic mechanisms after catalytic-class-specific templates exist

## 5. Inputs

### 5.1 Protein identity input

Use protein embedding（蛋白嵌入表示）from an ESM-style model or another sequence/structure model.

```text
sequence / structure
→ h_protein
```

### 5.2 GS complex input

Use a modeled or known substrate-bound state.

```text
protein + substrate bound state
→ h_GS
```

The GS complex can be produced by experimental structures, docking, homology modeling, PLACER, AlphaFold-style complex prediction, or other external workflows.

### 5.3 Reaction prior input

Reaction prior（反应先验）is mandatory for Project 02 because the model should know which chemistry is being evaluated.

Possible reaction prior inputs:

```text
reaction template ID
catalytic class
reaction step
atom mapping
product structure
bond-breaking / bond-forming pattern
proton transfer pattern
required catalytic residues
expected Δq pattern
known TS template, if available only during training or as a non-inference reference
```

### 5.4 TS prediction model output

The key difference from Project 01 is that Project 02 does not consume a true TS at inference. Instead, it consumes a predicted TS prior:

```text
GS + product + atom mapping / reaction template
→ TS prediction model
→ predicted TS geometry / predicted TS embedding / predicted Δq / predicted TS-like state
```

This output can be used directly as an embedding or passed into PLACER for conformer generation/screening.

### 5.5 PLACER screening input

Use PLACER to generate or screen conformers compatible with the predicted TS prior:

```text
protein + GS + predicted TS prior
→ PLACER-screened predicted-TS-compatible ensemble
→ h_pred_TS_ensemble, ensemble features
```

This captures:

- side-chain flexibility（侧链柔性）
- ligand pose diversity（配体姿态多样性）
- predicted-TS-compatible geometry frequency（预测 TS 兼容构象比例）
- productive conformation probability（有效反应构象概率）

## 6. Training modes

### Mode A: true TS available during training

Use Project 01 data:

```text
GS + true/refined TS + QM/MM label
→ supervised true-TS embedding and teacher outputs
```

Training signals include:

```text
ΔG‡_QMMM
ΔΔG‡_QMMM
Project01_teacher_ΔG‡
Project01_teacher_S_field
Project01_teacher_S_geometry
Project01_teacher_S_ensemble
true_TS_embedding
```

### Mode B: no true TS at inference

At inference:

```text
GS + reaction prior
→ TS prediction model
→ predicted TS embedding/geometry
→ PLACER screening
→ predicted-TS student
→ ΔG‡ proxy / catalytic potential ranking
```

True TS coordinates must not be supplied to the inference pipeline.

Primary outputs:


Optional auxiliary outputs:


Do not make `kcat`, `KM`, or `kcat/KM` the first-stage primary target. They can be included later as optional external validation fields.

## 10. Training strategy

### Stage 1: teacher-student distillation from Project 01

Use Project 01 exports as training labels:

```text
true-TS teacher barrier prediction
QM/MM or DFT ΔG‡ / ΔΔG‡
S_field
S_geometry
S_ensemble
true_TS_embedding
```

Train Project 02 from:

```text
protein embedding + GS embedding + reaction prior + predicted TS embedding + PLACER-screened ensemble features
```

### Stage 2: predicted-TS alignment

Train the predicted TS representation to match Project 01 true-TS representations when true TS is available during training:

```text
L_TS_align = distance(z_TS_pred, z_TS_true)
```

This should be auxiliary. The final inference pipeline still uses predicted TS only.

### Stage 3: within-catalytic-class ranking

Use pairwise ranking within the same catalytic type or reaction template:

```text
variant A lower computed barrier than variant B
```

### Stage 4: optional experimental validation

Use experimental labels only when they are comparable and well annotated:

```text
relative activity
known beneficial mutation
kcat / KM / kcat/KM with assay metadata
```

Experimental data should not override the first-stage computed barrier objective.

## 11. Loss functions

Recommended multi-task losses:

```text
L_total =
  λ_barrier * L_barrier
+ λ_delta   * L_delta_delta
+ λ_rank    * L_rank
+ λ_distill * L_teacher_distill
+ λ_align   * L_TS_embedding_alignment
+ λ_field   * L_field_proxy
+ λ_geom    * L_geometry_proxy
+ λ_ens     * L_ensemble_proxy
```

Where:

- `L_barrier`: regression on QM/MM/DFT `ΔG‡` or teacher-predicted `ΔG‡`.
- `L_delta_delta`: regression on `ΔΔG‡` vs WT/reference.
- `L_rank`: pairwise ranking loss within catalytic class or reaction template.
- `L_teacher_distill`: student output matches Project 01 true-TS teacher output.
- `L_TS_embedding_alignment`: predicted TS embedding aligns with true TS embedding during training only.
- `L_field_proxy`: distillation from Project 01 reaction-projected electrostatic score.
- `L_geometry_proxy`: distillation from Project 01 catalytic geometry score.
- `L_ensemble_proxy`: distillation from Project 01 productive conformer fraction.

## 12. Evaluation

Evaluate four levels of generalization:

### Level 1: within-enzyme mutant prediction

Train and test on variants of the same enzyme, with variant-level splits.

### Level 2: within-family transfer

Train on some members of a family and test on held-out family members.

### Level 3: within-catalytic-class transfer

Example:

```text
train: PETase + CALB + esterase variants
validation: cutinase-like hydrolase
held-out test: another serine hydrolase family
```

### Level 4: cross-catalytic-class exploration

Only after class-specific templates are mature:

```text
serine hydrolase → metal hydrolase
hydrolase → redox enzyme
```

Metrics:

```text
Spearman correlation with computed ΔG‡ / ΔΔG‡
pairwise ranking accuracy
top-K enrichment for low-barrier candidates
leave-one-enzyme-out performance
leave-one-family-out performance
leave-one-reaction-template-out performance
performance gap to Project 01 true-TS teacher
```

## 13. Required baselines and controls

Project 02 must compare:

1. **GS-only baseline**: protein + GS, no TS prior.
2. **reaction-prior-only baseline**: reaction template without predicted TS structure/embedding.
3. **embedding-only baseline**: protein embedding only.
4. **predicted-TS student**: predicted TS embedding/geometry + PLACER screening.
5. **true-TS teacher upper bound**: Project 01 model with true TS input.
6. **wrong-TS control**: incorrect, shuffled, or mismatched TS prior.
7. **shuffled reaction template control**: incorrect reaction prior.

Key diagnostic question:

```text
Does predicted TS + PLACER screening close the gap between GS-only and true-TS teacher?
```

If wrong-TS or shuffled-reaction controls do not reduce performance, the model is not truly using reaction-state information.

## 14. Compute plan

### GPU tasks

- protein embedding generation
- TS prediction model inference
- PLACER screening / ensemble encoding
- neural model training
- teacher-student distillation

### CPU / external tasks

- reaction template preprocessing
- predicted TS feature extraction
- classical electrostatic proxy calculations
- optional QM/MM calibration for held-out predicted-TS candidates

Project 02 should minimize expensive QM/MM usage. It should mainly consume Project 01 exports and use additional QM/MM only for calibration or validation.

## 15. Deliverable

A working Project 02 implementation should produce:

```text
input enzyme + GS complex + reaction prior
→ predicted TS embedding/geometry
→ PLACER-screened predicted-TS-compatible ensemble
→ ΔG‡ proxy / ΔΔG‡ proxy
→ catalytic potential ranking
→ mechanism components
→ comparison against GS-only, wrong-TS, and true-TS teacher baselines
```

The key scientific claim should be:

> Within a catalytic class, predicted TS priors and PLACER screening can transfer true-TS barrier-learning signals from Project 01, enabling practical computed barrier proxy prediction when true TS coordinates are unavailable at inference.

# Project 02: General enzyme prediction model

Created: 2026-06-28

## 1. Project goal

This project builds a **general enzyme activity prediction model（通用酶活性预测模型）** that can estimate reaction potential across enzyme systems.

Unlike Project 01, this project should not require a manually supplied TS complex as a mandatory inference input.

The target setting is:

```text
enzyme sequence/structure + substrate/GS complex + reaction prior
→ predict ΔG‡ proxy or activity ranking
```

Here:

- GS（ground state，基态）is usually available or can be modeled as a substrate-bound enzyme complex.
- TS（transition state，过渡态）may be available during training, but should be optional at inference time.
- Reaction prior（反应先验）can include reaction template, atom mapping, product, bond-breaking/bond-forming pattern, or a TS embedding generated by a TS prediction model.

## 2. Relationship to Project 01

Project 01 is a fixed-reaction, TS-aware evaluator.

Project 02 is the generalization layer.

The intended transfer path is:

```text
Project 01 learns mechanism-conditioned TS-GS scoring
        ↓
Distill physical components and labels
        ↓
Project 02 learns broader enzyme activity prediction
```

Project 02 should reuse the physical decomposition from Project 01, but avoid depending on explicit DFT TS coordinates at inference time.

## 3. Scientific hypothesis

A general enzyme predictor can be made more data-efficient if activity is not treated as a black-box label.

Instead, activity should be decomposed into:

```text
1. protein identity / mutation context
2. GS binding and ligand pose
3. predicted or encoded TS prior
4. geometry feasibility
5. electrostatic TS stabilization proxy
6. conformational ensemble accessibility
7. strain / product-release penalties
```

This is not a pure `embedding → activity` model. It is a reaction-aware prediction model.

## 4. Inputs

### 4.1 Protein identity input

Use protein embedding（蛋白嵌入表示）from an ESM-style model or another general protein sequence/structure model.

```text
sequence / structure
→ h_protein
```

### 4.2 GS complex input

Use a modeled or known GS enzyme-substrate complex.

```text
protein + substrate bound state
→ h_GS
```

### 4.3 PLACER ensemble input

Use PLACER-style conformational ensembles to represent accessible ligand and side-chain states.

```text
GS conformer ensemble
→ h_ensemble
```

This captures:

- side-chain flexibility（侧链柔性）
- ligand pose diversity（配体姿态多样性）
- productive conformation frequency（有效反应构象比例）

### 4.4 Reaction prior input

This is the key difference from a plain protein activity predictor.

Possible reaction prior inputs:

```text
reaction template
atom mapping
product structure
bond-breaking/bond-forming pattern
known TS template, if available
TS embedding generated by a TS prediction model
predicted Δq pattern, if available
```

## 5. Optional TS encoder

A TS encoder（过渡态编码器）can be added to generate a reaction embedding.

For Project 02, the TS encoder should support two modes:

### Mode A: known TS available during training

```text
known GS + known TS
→ z_TS_supervised
```

Use this for supervision and distillation.

### Mode B: TS not explicitly available at inference

```text
GS + product + atom mapping / reaction template
→ predicted z_TS
```

This allows the model to use a TS prior without requiring a fully optimized TS complex.

## 6. Recommended architecture

```text
Protein sequence / structure
        ↓
Protein encoder, e.g. ESM
        ↓
h_protein

GS complex + PLACER ensemble
        ↓
State / ensemble encoder
        ↓
h_GS, h_ensemble

Reaction template / product / optional TS model
        ↓
Reaction or TS encoder
        ↓
z_reaction or z_TS

Fusion
        ↓
reaction-aware activity head
        ↓
ΔG‡ proxy / activity ranking
```

## 7. Model outputs

Primary outputs:

```text
ΔG‡ proxy
activity ranking
kcat/KM proxy
```

Optional auxiliary outputs:

```text
predicted TS-like geometry shift
predicted Δq pattern
predicted reaction-projected electric-field score
productive conformer probability
```

## 9. Training strategy

### Stage 1: supervised distillation from Project 01

Use Project 01 outputs as labels:

```text
known TS-aware score
ΔG‡
S_field
S_geometry
S_ensemble
```

Train Project 02 to predict these labels from:

```text
protein embedding + GS ensemble + reaction prior
```

### Stage 2: activity ranking

Use experimental enzyme labels where available:

```text
kcat
KM
kcat/KM
relative activity
```

Convert to relative barrier when possible:

```text
ΔΔG‡ = -RT ln(activity_mut / activity_WT)
```

### Stage 3: cross-reaction generalization

Train on several reaction families and evaluate leave-one-family-out.

Example split:

```text
train: CALB + PETase + nylonase
validation: PUase
future test: laccase or carbonic anhydrase
```

## 10. Loss functions

Recommended multi-task losses:

```text
L_total =
  λ_barrier * L_barrier
+ λ_rank    * L_rank
+ λ_field   * L_field_proxy
+ λ_geom    * L_geometry_proxy
+ λ_ens     * L_ensemble_proxy
```

Where:

- `L_barrier`: regression on ΔG‡ or ΔΔG‡
- `L_rank`: pairwise ranking loss
- `L_field_proxy`: distillation from Project 01 reaction-projected field score
- `L_geometry_proxy`: catalytic geometry feasibility supervision
- `L_ensemble_proxy`: productive conformer fraction supervision

## 11. Evaluation

Evaluate three levels of generalization:

### Level 1: within-enzyme mutant prediction

Train and test on variants of the same enzyme family.

### Level 2: within-reaction family transfer

Train on one hydrolase family and test on another hydrolase.

### Level 3: cross-reaction transfer

Train on hydrolases and test on redox or metalloenzyme systems.

Metrics:

```text
Spearman correlation
pairwise ranking accuracy
top-K enrichment
leave-one-family-out performance
ability to recover known beneficial mutations
```

## 12. Compute plan

### GPU tasks

- protein embedding generation
- PLACER-style ensemble encoding
- neural model training
- TS encoder inference, if implemented

### CPU tasks

- optional QM/MM calibration
- reaction template preprocessing
- classical electrostatic proxy calculations

Project 02 should minimize expensive QM/MM usage. It should consume distilled signals from Project 01 whenever possible.

## 13. Deliverable

A working Project 02 implementation should produce:

```text
input enzyme + GS complex + reaction prior
→ predicted ΔG‡ proxy
→ activity ranking
→ optional decomposition into geometry / field / ensemble components
```

The key scientific claim should be:

> General enzyme prediction improves when protein embeddings are conditioned on reaction-state priors and constrained by mechanism-aware scores, rather than trained as direct black-box activity predictors.
